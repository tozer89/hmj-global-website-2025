Guided setup

Build the forecast

Work through the steps in order. Basic mode keeps only the normal inputs in view; Advanced reveals receipt overrides, invoice timing, comparison setup, and edge-case controls.

Step 1

Account setup

Set the client, insured limit, opening balance, and reporting currency. These are the minimum account facts the whole forecast depends on.

Step 2

Terms and tax

These settings control how invoices turn into receipts. Opening-balance receipts reduce receivables already on the ledger; forecast-invoice receipts come from invoices created inside this forecast.

Step 3

How do you want to model growth?

Choose one modelling method only. The inactive method stays preserved for later but does not affect the live forecast.

This forecast is currently driven by Contractor mode

Contractor-based assumptions

Use this when you know the current headcount, the extra contractors to test, and the weekly commercial value per contractor.

Active

Advanced contractor inputs are optional. If you enter a direct per-contractor invoice value, it takes priority over weekly pay and margin for the active forecast.

Direct weekly uplift assumptions

Use this when you only want to test the extra weekly invoicing value, without modelling contractor headcount.

Inactive

Base run-rate invoice values are advanced controls. Leave them blank for a straightforward uplift-only check.

Step 4

Advanced timing & receipts

Open these controls only when you need to override normal timing assumptions, load a comparison scenario, or add known receipt lines.

Invoice timing and receipt logic Use this only when weekly invoicing defaults are not enough.

Manual receipt timing only matters if the default term-driven timing is not enough. Otherwise the engine will map invoice dates to receipts automatically.

Opening-balance receipts and weekly overrides Use manual dates only when you know expected collections from the opening receivables book.

Manual opening-balance receipts are only active when the opening-balance mode above is set to Manual. Weekly adjustments remain available as a separate override layer.

Comparison and notes Use comparison only when you want a side-by-side overlay.

Step 5

Calculate and share

Calculate is the primary action. Save, export, print, and GPT refresh stay secondary so the core workflow remains clear.
